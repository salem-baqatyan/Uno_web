# Uno WebApp - لعبة أونو لتيليجرام 🃏🔥

مشروع متطور يعيد تقديم لعبة الأوراق الشهيرة **Uno** كـ **WebApp** تفاعلي مدمج داخل بوتات التيليجرام. تم بناء اللعبة لتوفر تجربة بصرية كاملة، حيث يمكن للاعبين رؤية أوراقهم، السحب من المجموعة، ورمي الأوراق مع مؤثرات حركية، كل ذلك داخل نافذة واحدة في تطبيق التيليجرام.

## 🎯 الهدف من المشروع

توفير بديل رقمي وتفاعلي للعبة الأونو التقليدية داخل المجموعات، مع أتمتة كاملة لقوانين اللعبة (مثل الأوراق الخاصة، وتغيير الألوان، وتتابع الأدوار)، مما يسهل اللعب الجماعي عن بُعد دون تعقيد الأوامر النصية.

## ✨ المميزات التقنية (Technical Features)

### 1. نظام محرك اللعبة (Game Engine Logic)

* **توزيع الأوراق:** نظام عشوائي لتوزيع الأوراق على اللاعبين مع ضمان توازن اللعبة.
* **قوانين الأوراق الخاصة:** معالجة برمجية دقيقة لمهام الأوراق الوظيفية:
* **Skip:** تخطي اللاعب التالي.
* **Reverse:** عكس اتجاه اللعب.
* **Draw Two (+2) & Wild Draw Four (+4):** إضافة الأوراق آلياً للاعب التالي.
* **Wild Card:** واجهة لاختيار اللون المطلوب عند رمي الورقة الجوكر.


* **التحقق من الصلاحية:** منطق برمجى يمنع اللاعب من رمي ورقة لا تتطابق مع الورقة الموجودة على الطاولة (من حيث اللون أو الرقم).

### 2. واجهة المستخدم والتفاعل (UI/UX)

* **عرض يد اللاعب (Player Hand):** شريط تمرير أفقي سلس يستعرض أوراق اللاعب مع إمكانية اختيارها بسهولة.
* **الحركات (Animations):** تأثيرات بصرية عند سحب الأوراق أو رميها في المنتصف لتعزيز الواقعية.
* **تحديث الأدوار:** مؤشر بصري يوضح للاعب متى يحين دوره للعب.

### 3. تكامل Telegram WebApp API

* **التواصل مع البوت:** إرسال تحديثات اللعبة (مثل من فاز، ومن سحب ورقة) إلى البوت ليتم إعلانها داخل المجموعة.
* **التوافق التام:** تصميم واجهة مستخدم متجاوبة تتناسب مع أحجام الشاشات المختلفة وأوضاع الرؤية (Dark/Light).

## 🛠 التقنيات المستخدمة (Technical Stack)

* **JavaScript (ES6+):** لمعالجة المصفوفات المعقدة المسؤولة عن سطح اللعبة وأوراق اللاعبين.
* **HTML5 Canvas / CSS Transitions:** لإنشاء الرسوم المتحركة وحركة الأوراق الانسيابية.
* **Telegram WebApp SDK:** للربط الوثيق بين واجهة الويب وحساب المستخدم في تيليجرام.
* **Game State Management:** نظام لمتابعة حالة "اللون الحالي" و "الرقم الحالي" وتدفق اللعب.

## 📂 هيكلية المشروع (Project Structure)

```text
├── index.html        # الواجهة الرئيسية (منطقة اللعب ويد اللاعب)
├── css/style.css     # تصميم بطاقات الأونو وتأثيرات الحركة
├── js/uno_logic.js   # المحرك البرمجي للقوانين ومعالجة الأدوار
└── assets/           # صور البطاقات والمؤثرات الصوتية والبصرية

```

## 🚀 كيفية التشغيل والتجربة

1. **الاستضافة:** رفع المشروع على استضافة آمنة تدعم `HTTPS`.
2. **الربط بالبوت:** يتم فتح اللعبة عبر "بوت طويق" من خلال زر يرسل المستخدم إلى رابط الـ WebApp مع تمرير بيانات الجلسة.
3. **طريقة اللعب:** * يضغط اللاعب على الورقة المناسبة لرميها.
* في حال عدم وجود ورقة مناسبة، يضغط على "مجموعة السحب" للحصول على ورقة جديدة.



## 📝 ملاحظات التطوير

تعتبر لعبة الأونو من أكثر الألعاب تعقيداً من حيث المنطق البرمجي (Logic) نظراً لتعدد الحالات والقوانين، وقد تم تطوير هذا الـ WebApp ليكون بمثابة الإضافة الأقوى لـ "بوت طويق" لتعزيز التفاعل الاجتماعي داخل المجموعات.
